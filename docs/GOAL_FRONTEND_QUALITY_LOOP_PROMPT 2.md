제품 감성:
- Toss처럼 사소한 UI/UX까지 완성도가 있어야 한다.
- 단정하고 신뢰감 있고, 심플하지만 심심하지 않아야 한다.
- 사용자는 모바일 앱처럼 버튼이 적고, 다음 행동이 분명한 플랫폼을 기대한다.
- 글래스모피즘은 과하지 않게 사용한다.
  - 반투명 패널
  - 미세한 blur
  - 얇은 border
  - 깊이감 있는 shadow
  - 절제된 배경 레이어
- 단, 장식용 오브/과한 그래디언트/의미 없는 배경 효과에 의존하지 마라.
- 그래프와 데이터가 주인공이어야 한다.
- 차트, 탭 전환, 패널 열림/닫힘, 툴팁, 상태 변화에는 부드럽고 고급스러운 애니메이션을 넣어라.
- 한국어/영어 글씨체는 모두 Pretendard를 기본으로 통일하라.

사용자가 지적한 현재 문제:
- 배우기 탭에 배울 것이 너무 적다.
- 용어는 있지만 설명이 부족하다.
- 각 용어/항목의 첫 줄에 핵심 요약 한 줄이 필요하다.
- 각 항목은 최소 3줄 이상 설명해야 한다.
- 각 항목에는 시나리오 예시가 포함되어야 한다.
- 현재 버튼이 너무 많아 헷갈린다.
- 모바일 앱처럼 버튼이 적고 깔끔한 플랫폼이 되어야 한다.
- 애니메이션 효과가 적고 화면이 너무 평면적인 2D처럼 느껴진다.
- AI가 만든 화면처럼 보인다.
- 다른 우수한 서비스들의 UI/UX 완성도를 참고해 훨씬 더 자연스럽게 만들어야 한다.
- Pretendard 폰트를 한국어/영어 모두에 적용해야 한다.

프론트 구현 목표:
1. 전체 IA 재설계
   - 버튼 수를 줄인다.
   - 주요 액션만 남기고 나머지는 더보기/관리자/접힌 패널로 숨긴다.
   - 초보자가 "이제 무엇을 해야 하지?"라고 고민하지 않게 한다.

2. 배우기 탭 고도화
   - 각 용어 첫 줄에 핵심 요약 1문장을 표시한다.
   - 각 항목당 최소 3줄 이상의 쉬운 설명을 제공한다.
   - 각 용어에 초보자 시나리오 예시를 넣는다.
   - "왜 중요한가", "언제 확인해야 하나", "주의할 점", "예시 질문"을 분리한다.
   - 용어 설명은 카드 나열이 아니라 학습 경험처럼 구성한다.

3. 차트/그래프 중심 화면
   - 일봉/주봉/월봉 그래프를 메인 경험으로 만든다.
   - 차트는 비어 보이면 안 된다.
   - 주요 상승/하락 지점에는 마커를 표시한다.
   - 마커 호버 시 이유와 근거를 보여준다.
   - 매수 검토 구간, 분할매수 검토 구간, 관망 구간, 매도 검토 구간, 손절/리스크 관리 구간을 시각적으로 표현한다.
   - 직접적 수익 보장 표현은 피하고, 조건형/시나리오형 문구로 표현한다.

4. 매수/매도 판단 UX
   - 사용자는 언제 사고 팔아야 하는지 알고 싶어한다.
   - "지금 사라", "지금 팔아라"라고 단정하지 말고 아래처럼 표현한다.
     - "매수 검토 구간"
     - "분할매수 검토 구간"
     - "관망 구간"
     - "매도 검토 구간"
     - "리스크 관리 구간"
   - 각 구간에는 반드시 조건, 근거, 반대 신호, 신뢰도, 데이터 기준일을 표시한다.
   - 예시:
     - "20일선 회복과 거래량 증가가 함께 확인되면 매수 검토 구간으로 볼 수 있습니다."
     - "급등 후 거래량 감소와 긴 윗꼬리가 반복되면 일부 매도 검토 구간으로 볼 수 있습니다."
     - "전저점 이탈 시 리스크 관리 기준을 다시 세워야 합니다."

5. 모바일 수준 UX
   - 주요 액션은 1~3개만 보이게 한다.
   - 나머지 기능은 bottom sheet, overflow menu, segmented control, tabs, drawer 등으로 정리한다.
   - 모바일 390px에서 텍스트/버튼/차트/툴팁이 겹치지 않아야 한다.
   - 터치 영역은 충분해야 한다.

6. 애니메이션/인터랙션
   - 탭 전환, 패널 전환, 카드 hover, 차트 마커 hover, 버튼 press 상태에 부드러운 전환을 넣는다.
   - 과한 애니메이션은 금지한다.
   - 고급스럽고 빠른 느낌을 주는 micro-interaction을 사용한다.
   - skeleton/loading/empty/error state를 예쁘게 만든다.

7. Pretendard 적용
   - 한국어/영어 모두 Pretendard를 우선 사용한다.
   - 웹폰트 또는 시스템 설치 여부를 고려해 fallback을 둔다.
   - font-weight, line-height, letter-spacing을 정교하게 조정한다.
