# NEXT AI CHART USABILITY HARNESS PROMPT

## 목표

현재 구현 상태를 낙관적으로 보지 말고, 실제 화면과 코드 기준으로 부족한 점을 끝까지 찾아 개선한다.

특히 아래 문제를 우선 해결한다.

- 이동평균선이 차트에서 바로 이해되지 않는다.
- 언제 매수 검토, 관망, 매도 검토, 리스크 관리를 해야 하는지 차트에 충분히 드러나지 않는다.
- 초보자가 첫 화면만 보고 "이 서비스를 계속 쓰고 싶다"고 느끼기 어렵다.
- 1일, 1주, 1개월 버튼이 실제로 차트를 다시 그리고 있는지 검증해야 한다.

## 구현 요구사항

### 1. 차트 위 이동평균선 설명 강화

차트에 아래 내용을 직접 드러낸다.

- 5일선: 단기 흐름
- 20일선: 약 한 달 평균 흐름
- 60일선: 중기 흐름
- 현재가가 20일선 위인지 아래인지
- 20일선 위라고 무조건 좋은 것은 아님
- 20일선 아래라고 무조건 나쁜 것은 아님
- 거래량, 지지선, 저항선, 이벤트와 함께 봐야 함

가능하면 차트 위에 색상 범례를 둔다.

예시:

- 파란 선: 현재가
- 흰 선: 5일선
- 회색 점선: 20일선
- 노란 점선: 60일선

---

### 2. 매수/매도 검토 조건을 차트에 직접 노출

AI 카드 안에만 숨기지 말고, 차트 근처에 아래 조건을 보이게 한다.

- 매수 검토 조건
- 분할매수 검토 조건
- 관망 조건
- 매도 검토 조건
- 리스크 관리 조건

표현은 반드시 조건형으로 쓴다.

금지 표현:

- 지금 사라
- 지금 팔아라
- 무조건 매수
- 무조건 매도
- 수익 보장

허용 표현:

- 이 조건이 확인되면 매수 검토
- 이 반대 신호가 나오면 보류
- 이 지지선이 깨지면 리스크 관리
- 거래량이 동반되지 않으면 관망

---

### 3. 이벤트 호재/악재 이유 표시

이벤트 점이나 이벤트 설명 영역에 아래 내용을 표시한다.

- 호재 후보인 이유
- 악재 후보인 이유
- 중립 또는 확인 필요인 이유
- 반대 해석
- 근거 수준
- 출처 한계

뉴스나 공시 원문 확인이 부족하면 확정적으로 쓰지 않는다.

예시 문장:

- "가격 상승과 거래량 증가가 함께 나타나 호재 후보로 볼 수 있지만, 공시 원문 확인 전에는 확정 원인으로 보지 않습니다."
- "다음 거래일에 거래량 없이 종가가 밀리면 실패 돌파일 수 있습니다."

---

### 4. 1일, 1주, 1개월 버튼 검증

현재 `1월`처럼 보이는 버튼이 있다면 `1개월`로 바꾼다.

각 버튼 클릭 시 아래를 확인한다.

- `1일` 클릭 시 `interval=daily` 요청 발생
- `1주` 클릭 시 `interval=weekly` 요청 발생
- `1개월` 클릭 시 `interval=monthly` 요청 발생
- 클릭 후 차트 SVG가 비어 있지 않음
- 선, 밴드, 이벤트 점이 계속 렌더링됨
- 활성 버튼 표시가 정확함

---

### 5. 초보자 첫 화면 개선

첫 화면에서 초보자가 바로 이해할 수 있도록 최소한 아래 3가지를 보여준다.

- 지금 차트에서 가장 먼저 볼 것
- 매수 검토 전에 확인할 조건
- 반대 신호가 나오면 보류해야 하는 이유

예시:

```text
처음 볼 3가지
1. 현재가가 20일선 위인지 아래인지
2. 거래량이 평균보다 늘었는지
3. 저항선 근처에서 밀리는지
```
