## 0. 핵심 결론

현재 프론트는 `GOAL_FRONTEND_QUALITY_LOOP_PROMPT.md` 기준으로 완성 상태가 아니다.

특히 아래 문제 때문에 이전 보고서의 `495점 이상`, `완성 단계`, `지킴` 판단은 신뢰하기 어렵다.

- PC 화면에서 검색/AI 인사이트 영역 하단이 잘린다.
- 헤더의 `한국 주식 AI 리서치 / 오늘 시장을 초보자 언어로 읽는 브리프`가 좌측에 붙어 보인다.
- 첫 화면은 여전히 기존 구조 위에 패치를 얹은 느낌이 강하다.
- 사용자가 5초 안에 "이 서비스가 왜 좋은지"를 확실히 이해하기 어렵다.
- AI 기능이 제품의 핵심처럼 보이기는 하나, VC 관점에서 "진짜 AI/RAG 플랫폼"으로 보일 정도는 아니다.
- 그래프가 페이지의 메인이라는 요구를 완전히 충족하지 못한다.
- `FRONTEND_LOOP_STATE.md`는 상단에서 `지킴/완성 단계`라고 선언하지만, 하단 상세 체크리스트에는 미완료 항목이 대량으로 남아 있다.

따라서 다음 AI는 기존 화면을 조금 고치는 방식이 아니라, 프론트 IA, 레이아웃, 컴포넌트, CSS, 모션, 반응형, 검색, 차트, AI 패널을 전면 재설계해야 한다.

## 2. MD 보고 양식 기준 보고

### 2.1 최종 점수

현재는 최종 완료 점수가 아니라 재감사 점수다.

| 관점 | 점수 | 판정 |
|---|---:|---|
| 실제 사용자 | 315/500 | 서비스 가치는 보이나 복잡함, 잘림, 차트 중심성 부족 |
| 프론트 개발자 | 295/500 | 전면 재설계 미흡, PC clipping, motion/design system 부족 |
| 백엔드 개발자 | 380/500 | 기존 API와 데이터 기반은 있으나 검색/AI/RAG 계약이 제품 수준으로 덜 선명 |
| DevOps 개발자 | 385/500 | Docker/health 기반은 있으나 출시급 CI/CD/visual regression/운영 증거 부족 |
| 벤처캐피탈/주주 | 275/500 | AI 플랫폼으로서 차별점과 확장성이 첫 화면에서 강하게 드러나지 않음 |

495/500 기준을 넘은 관점이 없으므로 루프를 종료하면 안 된다.

### 2.2 프론트를 전면 재설계한 내용

현재 결과는 `전면 재설계`가 아니라 `기존 화면 위의 부분 개편`에 가깝다.

다음 루프에서는 `AppShell`, `HomeCommandCenter`, `UniversalSearch`, `MainChartWorkspace`, `AICopilotPanel`, `LearningDrawer`, `AdminConsole` 수준으로 새 IA를 설계해야 한다.

### 2.3 버튼을 줄인 방식

일부 관리자 기능은 뒤로 빠진 것으로 보이나, 사용자 화면에는 여전히 버튼, 링크, 접힘 버튼, 카드 액션이 많다.

다음 루프에서는 전면 액션을 아래 3개로 제한한다.

1. 검색하기
2. 차트 보기
3. AI에게 묻기

나머지는 탭, overflow menu, drawer, admin mode로 이동한다.

### 2.4 검색 기능 구현 내용

검색창과 검색 결과는 있으나 PC에서 결과 하단이 잘리고, 검색이 서비스의 핵심 인터랙션으로 완성됐다고 보기 어렵다.

다음 루프에서는 검색 결과를 hero 내부 absolute dropdown이 아니라 clipping이 없는 command palette 또는 dedicated results panel로 재설계한다.

### 2.5 AI 기능을 명확히 드러낸 방식

우측 AI 패널은 있으나 "여기가 AI 제품의 핵심"이라는 인상은 약하다.

다음 루프에서는 AI 패널을 다음 구조로 고정한다.

- AI 시장 해석
- AI 차트 판단
- AI 리스크 요약
- AI 질문 입력
- 근거/출처/신뢰도/한계

### 2.6 차트/매수/매도 검토 UX 개선 내용

현재 텍스트로 매수 검토, 분할매수, 관망, 매도 검토, 리스크 관리 조건은 보인다.

하지만 사용자가 기대한 것은 그래프 위에서 직접 구간과 이유를 보는 경험이다. 다음 루프에서는 차트 자체가 메인이 되어야 하고, 구간/마커/툴팁이 차트 위에서 작동해야 한다.

### 2.7 참고한 사이트 5곳 이상과 배운 점

현재 문서/보고에는 reference research가 충분히 실제 결과물로 연결됐다고 보기 어렵다.

다음 루프에서 다시 조사해야 하는 참고군:

1. Toss: 여백, 버튼 절제, 설명 밀도, 신뢰감
2. 고품질 핀테크 서비스: 금융 데이터의 안정감 있는 정보 구조
3. 투자/증권 앱: 차트와 주문/판단 영역의 위계
4. AI SaaS: AI 기능을 제품 핵심으로 보이게 하는 패널 구조
5. 데이터 대시보드: 검색, 필터, 차트, 인사이트의 한 화면 연결

### 2.8 기능/버튼별 테스트 결과

현재 테스트 결과는 충분하지 않다.

특히 아래 테스트가 필요하다.

- PC 1440px, 1280px, 1024px에서 검색 결과 잘림 없음
- 모바일 390px, 430px에서 검색/차트/AI 패널 겹침 없음
- `AI 인사이트` 클릭/입력/검색 결과 노출 시 clipping 없음
- 헤더 좌우 여백이 정상이고 화면 끝에 붙지 않음
- 모든 주요 버튼이 실제로 화면 전환 또는 의미 있는 동작 수행
- 차트 일봉/주봉/월봉 전환
- 마커 hover tooltip이 화면 밖으로 나가지 않음
- 배우기 용어 상세 expand/collapse

## 3. 세부 누락 사항

### 3.1 PC 잘림 문제

사용자가 지적한 `AI 인사이트` 검색 결과 잘림은 실제로 발생한다.

원인 후보는 다음 조합이다.

```css
.marketHero {
  overflow: hidden;
}

.searchResults {
  position: absolute;
  top: calc(100% + 8px);
}
```

검색 결과가 `.marketHero` 안에서 absolute로 펼쳐지는데, 부모 hero가 `overflow: hidden`이라 하단이 잘린다.

해결 방향:

- `.marketHero`의 `overflow: hidden` 제거 또는 `overflow: visible` 적용
- 장식 배경만 별도 pseudo-element로 숨기고 실제 콘텐츠 overflow는 허용
- 검색 결과를 hero 밖 portal/root layer로 이동
- desktop에서는 command palette, mobile에서는 bottom sheet로 분기
- 검색 결과 max-height와 scroll, z-index, viewport collision 처리 추가

### 3.2 헤더 좌측 붙음 문제

`한국 주식 AI 리서치 / 오늘 시장을 초보자 언어로 읽는 브리프`가 좌측에 붙어 보이는 문제도 UX 품질 문제다.

해결 방향:

- 앱 전체 shell에 일관된 max-width와 gutter 적용
- top bar와 main content의 좌우 정렬 기준 통일
- PC에서는 32~48px 이상의 안정적인 좌우 여백 유지
- wide desktop에서는 중앙 정렬된 content rail 사용
- 모바일에서는 20~24px gutter 유지

### 3.3 "기존 페이지에서 달라지지 않았다" 문제

현재 화면은 완전히 새 플랫폼이라기보다 기존 구조의 hero, card, chart, panel을 재배치한 느낌이다.

해결 방향:

- 기존 `App.jsx` 중심 구조를 page shell과 feature component로 분해
- 홈 화면을 `Command Center`로 재설계
- 첫 화면에서 달력/운영 도구 느낌 제거
- 차트, AI, 검색을 한 화면에서 서로 연결된 경험으로 구현

### 3.4 "어디가 AI인지 모르겠다" 문제

현재 AI는 텍스트 패널 하나처럼 보인다.

해결 방향:

- AI 인사이트를 카드가 아니라 주 기능 패널로 배치
- AI가 어떤 데이터를 읽었는지 retrieval/source chips 표시
- `차트에서 이상 신호 발견 -> AI가 이유 설명 -> 리스크/반대 신호 제시` 흐름을 눈에 보이게 연결
- VC 관점에서 "AI가 확장 가능한 엔진"임을 보여주는 evidence layer 추가
